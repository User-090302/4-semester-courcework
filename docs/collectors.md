
### 1. Назначение модуля

Модуль `collectors.py` реализует систему сбора данных о состоянии операционной системы. Он предоставляет набор классов, каждый из которых отвечает за получение определённого типа системной информации: CPU, RAM, сеть, диски и вентиляторы.

Основная цель модуля — периодический сбор метрик системы и предоставление актуальных данных другим частям приложения (в первую очередь веб-интерфейсу).



### 2. Роль в архитектуре системы

Данный модуль является ядром **data layer (слой сбора данных)** системы мониторинга.

Он выполняет следующие функции:
- взаимодействие с операционной системой через `psutil`;
- периодическое обновление метрик в отдельных потоках;
- подготовку данных для последующего отображения в веб-интерфейсе;
- частичную генерацию HTML на основе шаблонов (в рамках MVP).



### 3. Базовый класс collector



#### 3.1. Назначение

Класс `collector` является базовым абстрактным классом, от которого наследуются все конкретные collectors.

Он задаёт общий интерфейс и механизм периодического обновления данных.



#### 3.2. Атрибуты

- `bfG` — lambda-функция для перевода байтов в гигабайты  
- `bfM` — lambda-функция для перевода байтов в мегабайты  
- `_lastRun` — время последнего обновления данных  
- `_freq` — частота обновления (в секундах)  
- `data` — структура с именами/метаданными (опционально)  
- `dataD` — актуальные собранные данные  



#### 3.3. Методы



##### `__init__(NewFrequency)`
Инициализация collector:
- задаёт частоту обновления данных;
- вызывает `readTemplate()` (загрузка HTML-шаблонов);
- инициализирует структуру данных;
- выполняет первичный сбор данных.



##### `run()`
Основной цикл обновления данных:

- работает бесконечно;
- проверяет таймер обновления;
- вызывает `GetData()` при наступлении интервала `_freq`;
- обновляет внутреннее состояние `dataD`;
- выполняется в отдельном потоке.



##### `GetData()`
Абстрактный метод (переопределяется в наследниках).  
Возвращает актуальные данные конкретного collector’а.



##### `GetNames()`
Опциональный метод для получения метаданных (например, список дисков).



##### `returnData()`
Возвращает последнее сохранённое состояние данных (`dataD`).



##### `readTemplate()`
Абстрактный метод загрузки HTML-шаблонов (используется в конкретных collectors).



### 4. Реализация collectors



## 4.1. RamCollector



### Назначение
Сбор информации об оперативной памяти и swap-разделе.



### Особенности реализации
- использует `psutil.virtual_memory()`;
- использует `psutil.swap_memory()`;
- данные форматируются и сразу подставляются в HTML-шаблон;
- результат возвращается в виде готовой HTML-строки.



### Собираемые данные
- общий объём RAM
- используемая память
- свободная память
- swap total/used/free



## 4.2. CPUCollector



### Назначение
Сбор информации о процессоре, включая нагрузку по ядрам.



### Особенности реализации
- использует `psutil.cpu_percent(percpu=True)`;
- использует `psutil.cpu_freq(percpu=True)`;
- пытается получить имя CPU через системные команды (`wmic`, `/proc/cpuinfo`);
- поддерживает шаблон для отображения каждого ядра отдельно;
- формирует HTML-представление данных.



### Собираемые данные
- загрузка CPU по ядрам
- частота каждого ядра
- температура процессора (если доступна)
- модель CPU



## 4.3. NetCollector



### Назначение
Мониторинг сетевой активности интерфейсов.



### Особенности реализации
- измеряет скорость передачи данных через разницу двух снимков состояния;
- использует задержку `time.sleep(1)` для вычисления скорости;
- получает IP-адреса интерфейсов;
- формирует HTML-блок для каждого сетевого интерфейса.



### Собираемые данные
- входящая скорость
- исходящая скорость
- общий объём отправленных/полученных данных
- IP-адрес интерфейса



### Ограничение
- измерение скорости реализовано через blocking sleep, что влияет на поток выполнения.



## 4.4. DisksCollector



### Назначение
Сбор информации о дисках и файловых системах.



### Особенности реализации
- использует `psutil.disk_partitions()` для получения списка дисков;
- использует `psutil.disk_usage()` для анализа заполненности;
- использует `psutil.disk_io_counters()` для расчёта скорости чтения/записи;
- формирует структурированные данные по каждому диску.



### Собираемые данные
- общий объём диска
- использованное пространство
- свободное пространство
- скорость чтения/записи



### Особенности
- используется двойной snapshot с задержкой для вычисления скорости I/O;
- данные возвращаются как словарь, а не HTML (в отличие от некоторых других collectors).



## 4.5. FansCollector



### Назначение
Сбор информации о скорости вращения вентиляторов системы.



### Особенности реализации
- использует `psutil.sensors_fans()`;
- поддерживает несколько вентиляторов на одном устройстве;
- нормализует структуру данных в единый словарь;
- формирует HTML-представление через шаблон.



### Собираемые данные
- скорость вращения вентиляторов (RPM)
- идентификаторы вентиляторов



### Ограничения
- зависит от поддержки sensors в ОС/железе;
- может возвращать пустые данные на некоторых системах.



## 5. Общие особенности реализации модуля

- используется библиотека `psutil` как основной источник системных метрик;
- каждый collector работает автономно и обновляет данные независимо;
- часть collectors возвращает HTML напрямую (MVP-решение);
- присутствует смешение уровней: data layer + presentation logic.



## 6. Ограничения архитектуры

- смешение логики сбора данных и HTML-рендеринга;
- использование blocking sleep внутри collectors (в NetCollector и DisksCollector);
- отсутствие единого формата данных (dict vs HTML);
- отсутствие синхронизации доступа к `dataD` (потенциальный race condition);
- частичная зависимость от платформы (CPU temp, fans).
