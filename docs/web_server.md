### 1. Назначение модуля

Модуль `web.py` реализует веб-серверную часть системы мониторинга ресурсов. Он построен на базе Flask и отвечает за предоставление HTTP-интерфейса, а также генерацию HTML-страниц на основе шаблонов.

Основная функция модуля — отображение данных, собранных collectors, в виде веб-страницы.


### 2. Роль в архитектуре системы

Данный модуль выполняет роль **presentation layer (слой представления)** и обеспечивает:

- запуск HTTP-сервера;
- обработку веб-запросов;
- формирование HTML-ответов;
- интеграцию с ядром системы через наследование класса `WebServer` в core.py.


### 3. Функциональные компоненты


#### 3.1. Функция `rednderHtml(html: str, replaces={})`

Назначение:
Простая система шаблонизации HTML-страниц.

Принцип работы:
- принимает HTML-шаблон;
- выполняет замену плейсхолдеров вида `key → value`;
- возвращает готовую HTML-строку.

Особенности:
- используется примитивная строковая замена;
- не является полноценным шаблонизатором (вроде Jinja2);
- предназначена для MVP-реализации.


#### 3.2. Функция `loadHtml(path)`

Назначение:
Загрузка HTML-шаблонов из файловой системы.

Реализация:
- открывает файл по указанному пути;
- возвращает его содержимое как строку.


### 4. Класс WebServer


#### 4.1. Назначение класса

`WebServer` является базовым HTTP-сервером системы. Он инкапсулирует Flask-приложение и предоставляет интерфейс для расширения в ядре системы.


#### 4.2. Атрибуты класса

- `app` — экземпляр Flask приложения
- `host` — IP-адрес сервера
- `port` — порт сервера


#### 4.3. Методы класса



##### `__init__(self, host='0.0.0.0', port=80)`

Инициализация веб-сервера:

- создаёт Flask-приложение;
- сохраняет сетевые параметры;
- вызывает `service()` (логика расширения);
- вызывает `_setup_routes()` (регистрация маршрутов).



##### `service(self)`

Заглушка для расширения функциональности.

Предназначена для переопределения в наследуемых классах (например, в `core.py`), где добавляется логика интеграции с collectors и потоками.


##### `_setup_routes(self)`

Заглушка для регистрации HTTP маршрутов.

В базовом классе не реализована, так как маршруты определяются в наследнике.

##### `getStatus(self)`

Возвращает текущий статус веб-сервера в виде словаря:

- статус работы;
- хост;
- порт.

Используется для диагностики и отладки.

##### `processData(self, data)`

Простая функция обработки входящих данных.

Фактически является заглушкой для будущего API-слоя.


##### `run(self)`

Запускает Flask-сервер:

- привязывает host и port;
- отключает debug-режим;
- запускает основной цикл обработки HTTP-запросов.


### 5. Особенности реализации

- Используется Flask как минимальный веб-фреймворк;
- отсутствует полноценный API слой (реализовано через заглушки);
- HTML генерируется вручную через строковые операции;
- класс спроектирован как базовый для расширения в core.py;
- архитектура ориентирована на MVP без шаблонизаторов и REST-структуры.


### 6. Ограничения

- отсутствует полноценная маршрутизация API;
- нет системы шаблонов (используется ручная замена строк);
- `rednderHtml` содержит упрощённую реализацию без экранирования;
- нет обработки ошибок HTTP;
- отсутствует разделение между API и UI логикой;
- функции `service` и `_setup_routes` не реализованы в базовом классе.
